‚öñÔ∏è Greedy Applicability Overview

‚úÖ Where Greedy Succeeds

  • Fractional Knapsack ‚Äî ratio-based selection is provably optimal
  • Job Scheduling ‚Äî profit-sorted slot filling maximizes earnings
  • Huffman Coding ‚Äî greedy merge produces optimal prefix codes
  • MST (Kruskal/Prim) ‚Äî greedy edge selection yields minimum weight tree

‚ùå Where Greedy Fails

  • 0/1 Knapsack ‚Äî greedy by ratio misses optimal combinations
  • Travelling Salesman ‚Äî nearest-neighbor gives suboptimal tours
  • Graph Coloring ‚Äî greedy may use more colors than needed
  • Subset Sum ‚Äî greedy largest-first can miss valid subsets

üìö Greedy Algorithm Theory

A greedy algorithm builds a solution piece by piece, always choosing the next piece that offers the most immediate benefit (locally optimal choice), hoping this leads to a globally optimal solution.

üí° Key Principle: Make the best choice at each step without reconsidering previous choices.

When does it work?

  • The problem has optimal substructure ‚Äî optimal solution contains optimal sub-solutions
  • The problem satisfies the greedy choice property ‚Äî a locally optimal choice leads to a globally optimal solution

To prove greedy works:

  • Exchange argument: Assume an optimal solution O that doesn't match the greedy solution G. Show you can swap a choice in O with the greedy choice without worsening the solution.
  • Stays ahead: Show that at every step, greedy's partial solution is at least as good as any other algorithm's partial solution.
üìù Example (Fractional Knapsack): If item with highest ratio isn't in optimal, swap it in ‚Äî the ratio guarantees equal or better profit per unit weight.

A problem has optimal substructure if an optimal solution to the whole problem contains optimal solutions to its sub-problems.

  • Knapsack: After deciding on item i, the remaining problem is a smaller knapsack
  • MST: After adding the cheapest safe edge, the remaining problem is MST on the contracted graph
  • Huffman: After merging two lowest-frequency nodes, the remaining problem is Huffman on n-1 symbols

Greedy fails when locally optimal choices don't combine into a globally optimal solution:

  • 0/1 Knapsack: Taking the highest-ratio item may block better combinations
  • TSP: Nearest-neighbor heuristic misses shorter global tours
  • Coin Change (non-standard): Denominations like {1,3,4} ‚Äî greedy picks 4+1+1=3 coins for amount 6, but optimal is 3+3=2 coins
‚ö†Ô∏è Always verify both greedy choice property AND optimal substructure before using greedy!
